***Для проверяющего***

Поставленную задачу я решил через бесконечный цикл while, позволяющий пользователю ввести в пустой массив столько строк, столько ему нужно (но не более 10 тысяч).
Цикл прерывается при введении ключевого слова quit.
После завершения бесконечного цикла переменная check меняется на единицу, что запускает второй цикл while, в котром по очереди проверяются строки составленного пользователем массива на предмет состава из 3-х и менее символов, после чего нужные строки добавляются в новый массив и выводятся на экран через пробел.
При написании кода возникла ошибка NullReferenceException, которая не мешала работе программы, но мазолила глаза и выглядела не красиво, потому я убрал её через комбинацию работы с исключениями try/catch, + решил добавить вывод на экран пояснения для пользователя и команду Console.Clear() для того что бы постоянно не чистить терминал вручную.


***Для пользователя***

**Доброго времени суток Пользователь!**

Перед тем как использовать эту простейшую программу прочти несколько правил:

1. Для запуска программы введи в консоль фразу "dotnet run".
2. Строки вводятся по очереди, когда вы закончили писать строку, добавляемую в массив, нажмите "Enter" и приступайте к написанию следующей.
3. Когла вы захотите выйти из программы и вывести результат на экран, нажмите "quit" строчными английскими буквами.
4. Массивы не могут содержать в себе больше 10 000 строк.
5. Для формирования нового массива программу придется перезапустить способом, описанным в пункте 1.
6. Не удаляйте и не меняйте строки кода, иначе программа сломается, для работы используйте консоль.
7. Приятного пользования =)
